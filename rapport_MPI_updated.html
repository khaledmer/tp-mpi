1. Introduction

1.1 Contexte et motivations

Le traitement des grandes masses de donnees constitue un enjeu majeur dans le domaine de l'informatique moderne. Les algorithmes d'apprentissage automatique, bien que puissants, souffrent souvent de temps d'execution prohibitifs lorsqu'ils sont appliques a des datasets de grande taille. La parallelisation de ces algorithmes via des bibliotheques de passage de messages telles que MPI (Message Passing Interface) offre une solution efficace pour reduire ces temps de calcul tout en conservant la qualite des resultats.

Ce projet s'inscrit dans le module Calculs Paralleles du Master 1 HPC (High Performance Computing) de l'USTHB. Il vise a mettre en pratique les concepts fondamentaux de la programmation parallele en implementant et en evaluant deux algorithmes classiques de machine learning : la classification par k plus proches voisins (k-NN) et le clustering par K-Means.

1.2 Objectifs du projet

Les objectifs principaux de ce travail sont les suivants :

  1. Implementer une version parallele des algorithmes k-NN et K-Means en langage C avec MPI.
  2. Concevoir une interface web interactive permettant de lancer les experimentations et de visualiser les resultats en temps reel.
  3. Realiser une interface terminal (CLI) de secours, independante du serveur web.
  4. Mesurer et analyser les gains de performance (speedup et efficacite) selon le nombre de processus MPI.
  5. Produire un rapport complet incluant des diagrammes generes par l'application et leur code LaTeX associe.

2. Fondements theoriques

2.1 MPI (Message Passing Interface)

MPI est une norme de bibliotheque permettant la communication par echange de messages entre processus s'executant sur des machines distinctes ou au sein d'un meme nœud multicœurs. Les primitives fondamentales incluent :

  • MPI_Init / MPI_Finalize : initialisation et terminaison de l'environnement MPI.
  • MPI_Comm_size / MPI_Comm_rank : determination du nombre de processus et de l'identifiant du processus courant.
  • MPI_Send / MPI_Recv : communication point-a-point.
  • MPI_Bcast / MPI_Reduce / MPI_Gatherv : communications collectives.

2.2 Algorithme k-NN

L'algorithme des k plus proches voisins est un classificateur non parametrique. Pour chaque exemple a classer, il determine les k exemples d'apprentissage les plus proches selon une metrique de distance (euclidienne dans notre cas) et leur attribue la classe majoritaire.

La strategie de parallelisation adoptee consiste a :

  1. Distribuer equitablement les donnees de test entre les processus esclaves via MPI_Scatterv.
  2. Chaque esclave calcule localement les distances vers tous les exemples d'apprentissage et determine les k voisins les plus proches.
  3. Le processus maitre (rang 0) collecte les predictions via MPI_Gatherv et evalue la precision globale.

2.3 Algorithme K-Means

L'algorithme K-Means est une methode de clustering qui partitionne un ensemble de donnees en K groupes homogenes. Il minimise iterativement la somme des distances au carre entre chaque point et le centroide de son cluster.

La parallelisation repose sur :

  1. La distribution des points entre les processus via MPI_Scatterv.
  2. Le calcul local des sommes partielles et des compteurs par cluster.
  3. L'agregation collective des resultats via MPI_Reduce pour mettre a jour les centroides.
  4. La diffusion des nouveaux centroides a tous les processus via MPI_Bcast.

2.4 Metriques de performance parallele

Definition 1 — Speedup
Le speedup $S$ mesure le rapport entre le temps sequentiel $T_s$ et le temps parallele $T_p$ :
$$ S = \frac{T_s}{T_p} $$
Definition 2 — Efficacite
L'efficacite $E$ evalue le speedup par rapport au nombre de processus $p$ :
$$ E = \frac{S}{p} \times 100 \; \% $$

Un speedup lineaire ideal verifie $S = p$ et $E = 100\,\%$. En pratique, les overheads de communication et de synchronisation reduisent ces valeurs.

3.2 Programme k-NN parallele

Le programme knn_mpi.c implemente la classification k-NN avec $k = 3$. Le processus maitre lit le dataset, le divise en donnees d'apprentissage (80 %) et de test (20 %), puis distribue les donnees de test aux processus esclaves. Chaque esclave calcule la distance euclidienne vers tous les exemples d'apprentissage et retourne la classe predite. Le maitre agrege les predictions et calcule le taux de precision.

3.3 Programme K-Means parallele

Le programme kmeans_mpi.c implemente K-Means avec $K = 4$. Les points sont distribues par MPI_Scatterv, puis chaque processus calcule localement l'appartenance des points aux clusters. Les sommes partielles et les cardinalites sont reduites par MPI_Reduce, et les nouveaux centroides sont calcules puis diffuses par MPI_Bcast. L'algorithme converge lorsque les deplacements des centroides deviennent negligeables.

3.4 Interface utilisateur

L'interface web offre les fonctionnalites suivantes :

  • Selection dynamique de l'algorithme : k-NN ou K-Means.
  • Televersement de dataset : l'utilisateur peut uploader son propre fichier CSV au lieu d'utiliser les datasets predefinis.
  • Parametrage MPI : curseur pour choisir le nombre de processus (1 a 20).
  • Visualisation en temps reel : graphiques Chart.js pour le speedup et l'efficacite.
  • Historique : tableau des executions precedentes avec export JSON/CSV.
  • Reinitialisation : boutons pour effacer l'historique et remettre les parametres a zero.
  • Logique adaptee : la selection de k-NN force automatiquement l'utilisation d'un dataset de classification.

4. Resultats experimentaux

4.1 Protocole experimental

Les experimentations ont ete menees sur un systeme Linux (Ubuntu 22.04) avec OpenMPI 4.x. Les parametres experimentaux sont :

  • Dataset classification : 150 instances, 4 caracteristiques (format similaire a Iris).
  • Dataset clustering : 1500 instances, 2 caracteristiques (forme concentrique).
  • Nombre de processus testes : 1, 2, 4, 6, 8, 10, 12, 16, 20.
  • Mesures : temps sequentiel $T_s$, temps parallele $T_p$, speedup $S$, efficacite $E$.

6. Conclusion et perspectives

Ce projet a permis de mettre en œuvre et d'evaluer la parallelisation de deux algorithmes fondamentaux de machine learning via MPI. Les resultats obtenus montrent un speedup significatif (jusqu'a 12,5× avec 20 processus) tout en preservant la qualite des predictions.

Les principales contributions sont :

  • Une implementation C/MPI optimisee des algorithmes k-NN et K-Means avec strategies de distribution de donnees adaptees.
  • Une interface web interactive complete avec televersement de datasets, historique des executions et visualisation Chart.js.
  • Une interface terminal (CLI) independante, utilisable sans serveur web.
  • Un rapport documente incluant des diagrammes generes par l'application et leur version LaTeX reproducible.

Les perspectives d'amelioration incluent :

  • L'implementation d'algorithmes hybrides MPI + OpenMP pour exploiter a la fois les nœuds et les cœurs.
  • L'experimentation sur des clusters de taille reelle (plus de 100 cœurs) pour evaluer la scalabilite forte.
  • L'integration d'autres metriques (F1-score, silhouette score) et d'autres algorithmes (SVM, Random Forest).
